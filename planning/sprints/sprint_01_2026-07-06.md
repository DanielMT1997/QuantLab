# Sprint 1 — Entorno reproducible y primeros datos

Periodo: 6–12 julio 2026

## Objetivo semanal

Construir un entorno mínimo de trabajo reproducible y descargar los primeros datos históricos para empezar el laboratorio cuantitativo.

El objetivo no es todavía ganar dinero, ni probar estrategias, ni optimizar nada. El objetivo es tener una base limpia sobre la que podamos investigar.

## Resultado esperado al final de la semana

Al terminar el sprint, el repositorio debe contener:

* Un entorno Python documentado.
* Un archivo `requirements.txt` funcional.
* Un notebook para descargar datos.
* Un archivo CSV con datos históricos diarios de SPY.
* Un notebook de exploración inicial.
* Cálculo básico de rendimientos.
* Un informe breve del sprint.

## Activo inicial

Usaremos **SPY** como primer activo.

Motivo: SPY es un ETF líquido que replica el S&P 500 y permite trabajar con precios ajustados, dividendos y un activo realmente negociable.

## Criterios de éxito

El sprint estará completado si puedo:

* Clonar el repositorio.
* Instalar las dependencias.
* Ejecutar los notebooks.
* Descargar datos históricos.
* Guardar los datos en `data/raw/`.
* Leer los datos desde un CSV.
* Dibujar el precio histórico.
* Calcular rendimientos diarios simples y logarítmicos.
* Escribir una conclusión breve sobre lo observado.

# Reglas del sprint

## Qué sí hacer

* Trabajar con calma.
* Hacer commits pequeños.
* Documentar lo aprendido.
* Priorizar que el código sea claro.
* Escribir conclusiones aunque sean simples.

## Qué no hacer todavía

* No probar estrategias.
* No optimizar parámetros.
* No usar machine learning.
* No operar con dinero real.
* No buscar señales complejas.
* No comparar veinte activos.
* No instalar diez librerías innecesarias.

## Regla para adelantar tareas

Si tengo más tiempo, puedo avanzar en este orden:

1. Completar la tarea base del día.
2. Completar la tarea extra del día.
3. Mejorar la documentación.
4. Adelantar la tarea base del día siguiente.
5. Parar y escribir una buena entrada de diario.

No puedo saltar a estrategias, machine learning, opciones, criptomonedas o modelos avanzados durante este sprint.

## Comandos Git recomendados

Al final de cada sesión:

```bash
git status
git add .
git commit -m "Sprint 1 - avance día X"
git push
```

Usar mensajes más específicos cuando sea posible, por ejemplo:

```bash
git commit -m "Add SPY data download notebook"
git commit -m "Compute daily returns for SPY"
git commit -m "Add sprint 1 report"
```
