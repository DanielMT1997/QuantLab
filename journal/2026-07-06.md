# Trabajo extra — Descarga de datos de SPY

## Objetivo

Crear el primer notebook del proyecto para descargar datos históricos diarios de SPY y guardarlos como archivo CSV.

## Trabajo realizado

He creado el notebook:

`notebooks/01_descarga_datos.ipynb`

Dentro del notebook he:

- importado `pandas`, `yfinance` y `Path`;
- definido el ticker `SPY`;
- definido la fecha inicial `1993-01-01`;
- creado la ruta de guardado de datos;
- descargado datos históricos diarios usando `yfinance`;
- inspeccionado las primeras y últimas filas del dataset;
- comprobado si había valores nulos;
- guardado los datos en formato CSV.

## Resultado

He conseguido crear correctamente un archivo CSV con datos diarios de SPY.

Las columnas principales son:

- `Open`: precio de apertura;
- `High`: precio máximo del día;
- `Low`: precio mínimo del día;
- `Close`: precio de cierre;
- `Adj Close`: cierre ajustado por dividendos, splits y otros ajustes;
- `Volume`: volumen negociado.

También he observado que `yfinance` puede devolver las columnas con una estructura jerárquica que incluye el ticker `SPY`. Esto no es un error, sino una forma de organizar datos cuando se descargan uno o varios activos.

## Dudas o aprendizajes

He aprendido la diferencia entre escribir código Python en un notebook y escribir comandos en la terminal.

También he aprendido que el CSV puede guardarse de dos maneras:

- como dato original en `data/raw/`;
- como dato limpio para análisis en `data/processed/`.

De momento, la decisión provisional es conservar los datos originales y trabajar después con una versión más limpia.

## Problemas encontrados

Al principio intenté ejecutar instrucciones de Python en la terminal de PowerShell, lo que produjo un error. La solución fue entender que:

- la terminal se usa para comandos del sistema;
- el notebook se usa para código Python.

También tuve que instalar o habilitar las extensiones de Python y Jupyter en VS Code para poder seleccionar el kernel del entorno virtual.
